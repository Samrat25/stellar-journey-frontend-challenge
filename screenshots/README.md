# Screenshots Guide

This directory contains screenshots for the README documentation.

## Required Screenshots

Please add the following screenshots to this directory:

### 1. `landing.png`
- **What to capture**: The landing page with hero section
- **URL**: http://localhost:5173/
- **Description**: Shows the beautiful hero section with "Get Started" button

### 2. `wallet-connected.png`
- **What to capture**: The connect page after successful wallet connection
- **URL**: http://localhost:5173/connect (after connecting)
- **Description**: Shows the wallet connection card with the full address displayed and "Connected" status

### 3. `balance.png`
- **What to capture**: The dashboard showing the balance card
- **URL**: http://localhost:5173/dashboard
- **Description**: Shows the XLM balance with the amount clearly visible (e.g., "9,999.99998 XLM")

### 4. `send-payment.png`
- **What to capture**: The send payment form filled out
- **URL**: http://localhost:5173/dashboard
- **Description**: Shows the send payment card with a destination address and amount entered

### 5. `transaction-success.png`
- **What to capture**: The success message after sending a payment
- **URL**: http://localhost:5173/dashboard (after successful transaction)
- **Description**: Shows the green success card with transaction hash and "View on Explorer" button

### 6. `history.png`
- **What to capture**: The transaction history page with transactions
- **URL**: http://localhost:5173/history
- **Description**: Shows the transaction history with at least 2-3 transactions listed

## How to Take Screenshots

### Using Browser DevTools (Recommended)

1. Open the app in your browser
2. Press F12 to open DevTools
3. Press Ctrl+Shift+P (Cmd+Shift+P on Mac)
4. Type "screenshot" and select "Capture full size screenshot"
5. Save the image to this directory

### Using Snipping Tool (Windows)

1. Press Windows + Shift + S
2. Select the area to capture
3. Save the image to this directory

### Using Screenshot Tool (Mac)

1. Press Cmd + Shift + 4
2. Select the area to capture
3. Save the image to this directory

## Image Requirements

- **Format**: PNG (preferred) or JPG
- **Resolution**: At least 1920x1080 for desktop views
- **Quality**: High quality, clear and readable
- **Content**: Should show the actual app running with real data

## Tips for Great Screenshots

1. **Use test data**: Make sure you have test XLM and some transactions
2. **Clean browser**: Hide bookmarks bar and extensions for cleaner look
3. **Zoom level**: Use 100% zoom for accurate representation
4. **Timing**: For transaction success, capture immediately after success message appears
5. **Consistency**: Try to use the same browser and window size for all screenshots
